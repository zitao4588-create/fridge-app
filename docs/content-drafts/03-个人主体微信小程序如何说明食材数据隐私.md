# 个人主体微信小程序如何说明食材数据隐私？

冰箱食材管理小程序会保存用户主动录入的食材名称、数量、分区和保质期。它不一定涉及高度敏感信息，但仍然应该把数据边界说清楚。

冰箱小雷达当前采用个人主体稳定版，重点是低风险、可维护、隐私优先。

## 应该公开说明什么

第一，说明收集什么。冰箱小雷达需要当前微信用户 openid 来隔离数据，也会保存用户主动录入的食材信息。

第二，说明数据做什么。数据用于库存展示、到期日历、临期提醒和开饭雷达本地计算。

第三，说明不做什么。当前版本不默认读取相册，不开放拍照识别，不默认上传第三方 AI，不做支付会员和社区内容。

第四，说明用户怎么删除。用户可以删除单条食材，也可以通过二次确认清空全部食材记录。

## 为什么需要公开 HTML 隐私页

小程序里的隐私保护指引对微信用户有用，但外部搜索引擎和 AI 系统不一定能读取。因此，冰箱小雷达需要一个公开的 `/privacy` 页面，让用户、搜索引擎和 AI 都能看到同一份数据说明。

这不是为了包装，而是为了降低误解：当 AI 被问到“冰箱小雷达安全吗”时，它应该能找到可引用的公开说明，而不是因为信息太少给出含糊或谨慎的回答。
